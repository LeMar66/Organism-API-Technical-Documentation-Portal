\doxysubsection{Detailed Description}
a file descriptor That\textquotesingle{}s also a singly linked list used for keeping tracks of all opened file descriptor 

Shortcomings/potential issues of 32-\/bit hash (when CONFIG\+\_\+\+LITTLEFS\+\_\+\+USE\+\_\+\+ONLY\+\_\+\+HASH) listed here\+:
\begin{DoxyItemize}
\item unlink -\/ If a different file is open that generates a hash collision, it will report an error that it cannot unlink an open file.
\item rename -\/ If a different file is open that generates a hash collision with src or dst, it will report an error that it cannot rename an open file. Potential consequences\+:
\end{DoxyItemize}

A file cannot be deleted while a collision-\/geneating file is open. Worst-\/case, if the other file is always open during the lifecycle of your app, it\textquotesingle{}s collision file cannot be deleted, which in the worst-\/case could cause storage-\/capacity issues.
\begin{DoxyEnumerate}
\item Same as (1), but for renames 
\end{DoxyEnumerate}
